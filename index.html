Readout

What discovery clarified

Reliability is the center of the project.

Caprock's stated priority is a reliable marketing data system. Dashboards are the useful surface, but the work succeeds or fails on trusted data, consistent definitions, and a process for checking accuracy.

  • Reporting needs to span channels. Paid, organic, web, email, calls, forms, MQLs, geo, and campaign views all need to line up.
  • The new dashboard should feel familiar. The difference is the cleaner data foundation underneath it.
  • Caprock's manual tracker exists because trust is missing. The replacement must make data easier to verify, not just easier to display.
  • Closed-loop ROI is a later maturity step. Dynamics matters, but the immediate need is to preserve source and campaign data so ROI can be connected cleanly later.

The requirements fall into four buckets.

  1. Trustworthy data and refresh cadence. The system needs reliable inputs, clear definitions, and a repeatable way to check accuracy.
  2. Flexible analysis. Caprock needs custom dates, period comparisons, campaign views, geo, audience, and channel filters.
  3. Consistent performance reporting. Paid, organic, website, calls, forms, email, SEO, and MQLs need to sit in one reporting frame.
  4. Audience-specific outputs. Marketing, leadership, board, and eventually regional teams need different views from the same reporting foundation.

Architecture

The proposed reporting foundation

Every delivery path uses the same foundation: collect the data, normalize it, validate it, then report from that trusted layer. This keeps the new dashboard focused on clear decisions instead of forcing it to solve every source, definition, and exception by hand.

1

Collect

Paid channels, organic channels, GA4, CallRail, forms, email, MQLs, and future CRM data.

2

Unify and validate

Shared campaign, channel, geo, date, and KPI definitions with an explicit quality check.

3

Report

New reporting views for marketing, leadership, channel detail, campaign detail, and board-ready summaries.

4

Extend

BigQuery, Squid, Dynamics, and closed-loop ROI can be added as the data model matures.

This is the foundation whether the project is delivered in one release or in phases. The choice now is delivery sequence.

Work chunks

Natural ways to phase value

These are the pieces that could be grouped into one release or sequenced as useful slices. The order should follow Caprock's most immediate pain.

Tracking and attribution foundation

Enabler

Clean up forms, calls, campaign fields, source capture, and key GA4 events before new reporting work inherits today's attribution gaps.

Start here if Caprock needs confidence in the inputs before investing in more reporting views.

Paid media performance readout

Fast visibility

Normalize paid media KPIs across Google Ads, LinkedIn, Simpli.fi, YouTube, and related channels, using the consistent metric set Caprock already requested.

Leads and MQLs by channel

Executive value

Bring form fills, calls, and Caprock's structured MQL data into the reporting view so leadership can see which channels are producing real lead activity.

Website, SEO, and organic visibility

Context layer

Fold in GA4 website performance, organic search, Google Business Profile, organic social, and related non-paid sources.

Campaign and geo breakdowns

Decision layer

Standardize campaign and geography fields so the team can filter across channels by the way Caprock actually plans and evaluates work.

Start here if the next decision is market allocation, regional performance, campaign mix, or spend by geography.

Executive and board reporting

Audience layer

Create higher-level summaries for leadership and board conversations, focused on leads, investment, trend, and what needs attention.

Delivery paths

Three reasonable ways to sequence it

Path A: one complete release

Build the system as a full package before Caprock adopts it.

  • Best for minimizing transition churn.
  • More complete at launch.
  • Longer time before the first usable deliverable.

Path B: phased release by priority

Our Recommendation

Pick the highest-value slices first, while keeping the same shared foundation underneath.

  • Best for getting useful reporting into hands sooner.
  • Lets Caprock validate direction before every later detail is locked.
  • Requires choosing which pain to solve first.

Path C: first release tied to a date

Shape the first slice around an upcoming meeting, board cycle, budget decision, or reporting deadline.

  • Best when a real date matters more than perfect completeness.
  • Can combine paid media, MQLs, or executive views depending on the meeting.
  • Anything outside that first deadline moves to the next release.

Working assumptions

What we will assume unless corrected

  • We are building a new dashboard experience. The reporting will sit on a cleaner, validated data layer from the start.
  • Daily refresh is the default. Faster refresh is a scoping detail if there is a specific operational need.
  • MQLs can begin as a structured manual input. Caprock's current process becomes cleaner before it becomes fully automated.
  • Dynamics is part of the path. Form and lead attribution should be captured in a way that can flow into CRM and support ROI later.

If any assumption is wrong, tell us now. Otherwise we will use these to prepare the next scope.
